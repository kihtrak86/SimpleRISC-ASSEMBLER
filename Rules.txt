SimpleRISC Assembler Rules

Please follow the rules below while writing SimpleRISC assembly programs.

1. Separate operands using spaces only

Do NOT use commas between operands.

Correct:
ADD R1 R2 R3

Incorrect:
ADD,R1,R2,R3

2. Load and Store instruction format

For LD and ST instructions, specify the base register first, followed by the immediate offset.

Correct:
LD R1 R2 5
ST R3 R4 8

Incorrect:
LD R1 5[R2]
ST R3 8[R4]

3. Label declaration

Labels must end with a colon (:).

Correct:
LOOP:

Incorrect:
LOOP

4. Place instructions below labels

Do NOT write an instruction on the same line as a label.

Correct:
LOOP:
ADD R1 R2 R3
SUB R4 R4 1

Incorrect:
LOOP: ADD R1 R2 R3

5. Do not place comments on the same line as a CMP instruction

Write the CMP instruction on its own line without any trailing comments.

Correct:
CMP R3 0

Incorrect:
CMP R3 0    # Compare R3 with 0

General Notes

* Instruction names are case-insensitive (ADD and add are both accepted).
* Register names should be written in the format R<number> (e.g., R0, R1, R10).
* Branch instructions (B, BEQ, BGT, CALL) must be followed by a valid label or address.
* Write only one instruction per line.
* Remove unnecessary spaces and blank lines for better readability.
